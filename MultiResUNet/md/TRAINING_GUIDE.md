# MultiResUNet Training Guide

## 训练结果分析与改进建议

### 当前问题分析

根据训练日志，发现以下问题：

1. **Dice 系数极低**：从 0.0687 降至 0.0000
2. **训练不稳定**：第 1 个 epoch 后有轻微学习，随后完全崩溃
3. **可能原因**：
   - 训练样本太少（仅 10 个样本）
   - Epoch 数量不足（仅 5 个）
   - 学习率可能不合适
   - 梯度爆炸/消失问题

### 已实施的改进

1. ✅ **修复评估指标计算** - 逐样本计算后取平均
2. ✅ **添加梯度裁剪** - 防止梯度爆炸
3. ✅ **添加学习率调度器** - 自动调整学习率
4. ✅ **增加训练样本** - 从 10 增加到 100
5. ✅ **增加训练轮次** - 从 5 增加到 50
6. ✅ **添加详细日志** - 记录训练历史和指标

### 建议的训练流程

#### 1. 数据验证（必须）
```bash
python diagnose_data.py
```
确保数据质量：
- ✓ 图像和掩码尺寸匹配
- ✓ 掩码是二值的（0/1）
- ✓ 通道数正确（4 通道）
- ✓ 图像值范围合理（归一化后 0-1）

#### 2. 开始训练
```bash
python train.py
```

#### 3. 监控训练过程
观察以下指标：
- **Loss 应该逐渐下降**
- **Dice 系数应该逐渐上升**（目标 > 0.7）
- **学习率应该适时调整**

### 超参数调优建议

如果训练效果仍不理想，尝试调整：

#### 学习率
- 初始学习率：`1e-4`（当前）
- 如果收敛太慢：尝试 `1e-3`
- 如果训练不稳定：尝试 `5e-5` 或 `1e-5`

#### Batch Size
- 当前：`2`（小样本情况）
- 如果显存充足：增加到 `4` 或 `8`
- 更大的 batch 通常更稳定

#### Epoch 数量
- 当前：`50`
- 如果未收敛：增加到 `100` 或 `200`
- 使用早停（early stopping）防止过拟合

#### 数据增强
如果训练集很小（< 100 样本），建议添加数据增强：
- 随机旋转（±15°）
- 水平/垂直翻转
- 随机缩放（0.8-1.2）
- 亮度/对比度调整

### 预期训练效果

使用 100 个训练样本，50 个 epoch：

**正常情况：**
- Epoch 1-10: Loss 快速下降，Dice 从 0.1 提升到 0.5
- Epoch 10-30: Loss 继续下降，Dice 从 0.5 提升到 0.7
- Epoch 30-50: 收敛，Dice 稳定在 0.7-0.85

**异常情况处理：**

1. **Dice 始终 < 0.1**
   - 检查数据标签是否正确
   - 检查图像和掩码是否匹配
   - 尝试更大的学习率

2. **Loss 不下降或震荡**
   - 降低学习率
   - 增加 batch size
   - 检查梯度是否爆炸

3. **训练集 Dice 很高但验证集很低**
   - 过拟合，需要数据增强或正则化
   - 增加训练数据
   - 使用 dropout 或 weight decay

### 模型保存和加载

训练完成后，模型会自动保存到 `models/` 目录：
- `model_architecture.pth` - 模型结构
- `model_weights.pth` - 模型权重

加载模型进行预测：
```python
import torch
from pytorch.MultiResUNet import MultiResUnet

model = MultiResUnet(input_channels=3, num_classes=4)
model.load_state_dict(torch.load('models/model_weights.pth'))
model.eval()
```

## 常见问题解答

**Q: 为什么我的 Dice 系数是 0？**
A: 可能原因：
- 数据加载错误（图像全黑）
- 标签格式错误
- 损失函数不匹配
- 学习率太高

**Q: 训练多久能看到效果？**
A: 通常 10-20 个 epoch 后应该能看到明显的 Dice 提升

**Q: 需要多少训练数据？**
A: 最少 50-100 样本，推荐 500+ 样本以获得好的泛化能力

**Q: 如何判断过拟合？**
A: 训练集 Dice 很高（>0.9）但验证集 Dice 很低（<0.5）
